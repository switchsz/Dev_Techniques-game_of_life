\doxysubsection{Detailed Description}
C\+Unit test case data type. \doxyref{C\+U\+\_\+\+Test}{p.}{struct_c_u___test} is a double linked list of unit tests. Each test has a name, a callable test function, and a flag for whether the test is active and thus executed during a test run. A test also holds links to the next and previous tests in the list, as well as a jmp\+\_\+buf reference for use in implementing fatal assertions.~\newline
~\newline


Generally, the linked list includes tests which are associated with each other in a \doxyref{C\+U\+\_\+\+Suite}{p.}{struct_c_u___suite}. As a result, tests are run in the order in which they are added to a suite (see \doxyref{C\+U\+\_\+add\+\_\+test()}{p.}{group___framework_gad9f198a8a5fa8cc6870c3c8be873869f}). ~\newline
~\newline


It is recommended that the name of each \doxyref{C\+U\+\_\+\+Test}{p.}{struct_c_u___test} in a suite have a unique name. Otherwise, only the first-\/registered test having a given name will be accessible by that name. There are no restrictions on the test function. This means that the same function could, in principle, be called more than once from different tests.
