Detailed Description

Error handling functions (user interface). CUnit uses a simple (and conventional) error handling strategy. Functions that can generate errors set (and usually return) an error code to indicate the run status. The error code can be inspected using the CU_get_error() function. A descriptive error message can be retrieved using CU_get_error_msg().
